# Magic Admin — 项目全局指引

## 项目概述

Magic Admin 是一个 React + TypeScript 管理后台，基于 Vite 构建，使用 pnpm 包管理。主要包含三大业务模块：**AI 管理**、**平台管理**、**企业管理**，以及横向的**能力管理**模块。

---

## 技术栈

| 层面 | 技术 |
|------|------|
| 框架 | React 18 + TypeScript |
| 构建 | Vite |
| UI 库 | Ant Design (antd) + 自定义组件层 (`components/`) |
| 样式 | antd-style (`createStyles`) |
| 路由 | React Router v6 (嵌套路由) |
| 状态管理 | Zustand (`src/stores/admin`, `src/stores/user`) |
| 数据请求 | ahooks `useRequest` + 自封装 `HttpClient` (基于 fetch) |
| 国际化 | i18next + react-i18next + 按命名空间自动发现 JSON |
| 包管理 | pnpm |
| 图标 | @tabler/icons-react |

## 核心开发模式

### 新增页面的标准流程

1. **路由常量** → `src/const/routes.ts` 的 `RoutePath` 和 `RouteName` 枚举新增条目
2. **权限常量** → `src/const/common.ts` 的 `PERMISSION_KEY_MAP` 新增权限 key，创建权限组数组，加入父级权限组
3. **路由配置** → 对应模块的 `routes.tsx`（如 `src/pages/AiManage/routes.tsx`），lazy 引入页面，配置 path/name/title/validate
4. **侧边栏菜单** → 对应模块的 `index.layout.tsx`，在 items 数组中新增菜单项（含 icon、validate、children）
5. **国际化菜单文案** → `src/locales/zh_CN/admin/common.json` 和 `en_US` 版的 `nav.*SubMenu.*`
6. **API** → `src/apis/constant.ts` 新增 URL，`src/apis/modules/*.ts` 新增方法，`src/apis/index.ts` 已自动通过工厂导出
7. **类型** → `src/types/*.ts` 在对应 namespace 中新增 interface
8. **页面级翻译** → `src/locales/zh_CN/admin/<domain>/<page>.json`（自动被 import.meta.glob 发现）
9. **页面组件** → `src/pages/<Module>/<Page>/index.page.tsx`，复杂页面拆 `components/` 子目录

### API 调用模式

```typescript
// 1. API 工厂（src/apis/modules/xxx.ts）
export const generateXxxApi = (client: HttpClient) => ({
  getList(params: Xxx.ListParams) {
    return client.post<Xxx.ListResponse>(RequestUrl.xxxList, params)
  },
})

// 2. 页面消费
const { XxxApi } = useApis()
const { data, loading, run } = useRequest(XxxApi.getList, { manual: true })
// 触发：run(params)
```

- HttpClient 底层基于 fetch，POST 自动 JSON.stringify
- 业务成功码为 `1000`，在响应拦截器中统一处理
- `useApis()` 从 `AdminProvider` 获取预配置的 client 实例

### 路由与菜单层级

```
一级菜单（TopMenu 顶部）: 平台 / AI / 企业
    └── 二级菜单（SideMenu 左侧分组标题）: 自定义大模型 / 模型管理 / 用量管控
         └── 三级菜单（SideMenu 子项）: 文本大模型 / 生图 / 模型管控
```

- 路由树在各模块 `routes.tsx` 中 export default 一个对象，由 `src/routes/routes.tsx` 聚合
- `validate(permissions, isSuperAdmin)` 控制菜单可见性和路由访问权限
- 超管判断：`MAGIC_PLATFORM_PERMISSIONS` / `MAGIC_ALL_PERMISSIONS` / `MAGIC_PERSON_PERMISSIONS`
- 每个模块的 Layout 组件使用 `SecondaryLayout`，传入 `items`（菜单配置）和 `openKeys`（默认展开）

### 权限体系

```
MAGIC_PLATFORM_PERMISSIONS (平台级) ⊃ MAGIC_ALL_PERMISSIONS (管理员级) ⊃ MAGIC_PERSON_PERMISSIONS (普通成员)
```

- 权限 key 格式：`admin.ai.xxx.query` / `admin.ai.xxx.edit` / `platform.ai.xxx.query`
- `admin.*` = 组织级，`platform.*` = 平台级，`workspace.*` = 工作区级
- 权限组数组（如 `AI_CUSTOM_MODEL`）聚合同类权限，再汇入总权限组（如 `AI_MANAGEMENT`）
- 页面中通过 `useRights` hook 或直接在 route validate 中校验

### 国际化

- 初始化：`src/locales/creator.ts` 中 `import.meta.glob` 自动发现 `./zh_CN/**/*.json` 和 `./en_US/**/*.json`
- 命名空间自动从文件路径提取：`./zh_CN/admin/ai/model.json` → namespace `admin/ai/model`
- 页面使用：`const { t } = useTranslation("admin/ai/model")`
- 菜单文案统一在 `admin/common` 命名空间的 `nav.*` 下
- 当前仅加载 zh_CN 和 en_US 两种语言

### 样式

- 使用 antd-style 的 `createStyles` 创建组件级样式
- 主题 token 通过 `token.magicColorUsages.*` 访问
- 避免内联样式，优先 createStyles

### 页面组件结构约定

```typescript
// index.page.tsx 标准结构
import { createStyles } from "antd-style"
import { useRequest, useMemoizedFn } from "ahooks"
import { useTranslation } from "react-i18next"
import { useApis } from "@/apis"

export default function XxxPage() {
  const { styles } = useStyles()
  const { t } = useTranslation("admin/xxx")
  const { XxxApi } = useApis()
  const { data, loading, run } = useRequest(XxxApi.method, { manual: true })
  // ...
}
```

## 各模块速查

### AI 管理 (`src/pages/AiManage/`)

- 路由定义：`src/pages/AiManage/routes.tsx`
- 布局菜单：`src/pages/AiManage/index.layout.tsx`（使用 SecondaryLayout）
- 权限组：`AI_MANAGEMENT` / `AI_CUSTOM_MODEL` / `AI_MODEL_MANAGEMENT` / `AI_USAGE_CONTROL`
- 子页面：ControlStrategy / FunctionConfig / ModelControl
- 复用平台侧页面：Model / ModelDetail / AIDrawing / ModelAuditLog / VideoModel

### 平台管理 (`src/pages/PlatformPackage/`)

- 路由定义：`src/pages/PlatformPackage/routes.tsx`
